Unity Workshop

Home Unity Workshop

Informatie

  • Datum:20/09/2017
  • Locatie:Corda 7: iClassroom
  • Uren:18

Dit seminarie bestond uit drie sessies en werd gegeven door twee bedrijven die in de gameindustrie aanwezig zijn. De eerste 2 sessies werden gegeven door Wouter Spaas van Firewolf Studios. De laatste sessie werd gegeven door Sam Agten, nu lector bij PXL, toen werkend voor LuGus Studios. Tijdens deze sessies hebben we gebruik gemaakt van Unity (een 3D game engine). Deze game engine kan gebruik maken van zowel C# code als Unity’s eigen versie van JavaScript, genaamd Unityscript. Voor deze sessies hebben we ons beperkt tot het schrijven van C#. Voor de C# code te schrijven hebben we gebruik gemaakt van Visual Studio. In Visual Studio hebben we de scripts aangemaakt die we met onze objecten moesten verbinden.

De bedoeling van de eerste twee sessies was om een basis Unity game te maken die wel iets weg had van het welbekende “Space invaders”. Eerst en vooral kregen we uitleg over het gebruik van Unity en hoe we de UI moesten opbouwen. Ook hebben we geleerd hoe we een model moesten designen en hoe we het rondvliegen, schieten en de vijanden moesten programmeren. De vijanden zouden dan verdwijnen wanneer ze geraakt werden door een kogel. Hier maakte we dus gebruikt van collision detection. We hadden zelf de vrije keuze in wat we precies deden op visueel vlak. Het design mochten we zelf wat bepalen.

Tijdens de tweede sessie was het de bedoeling om meer functionaliteit aan de game toe te voegen. Hier hebben we een menu met startscherm aan het spel toegevoegd, een scorebord, het aantal levens, etc.. Voor deze functionaliteiten toe te voegen moesten we een aantal scripts schrijven en deze linken aan de objecten. Hierna heb ik zelf nog wat zitten spelen met de modellen van de vijanden en hoe de omgeving er nu precies zou uitzien, waardoor ik een overtuigende achtergrond die de ruimte moest voorstellen heb kunnen toevoegen aan mijn spel.

Tot slot kregen we nog een demo hoe we een complex object moesten maken via Unity. Onze lesgever Wouter werkt in zijn vrije tijd aan een aantal objecten zoals een ruimteschip van Star Wars. Het was indrukwekkend hoe gedetailleerd deze objecten waren en hoeveel tijd hierin kruipt.

De laatste sessie werd gegeven door Sam Agten, die op dat moment werkte bij LuGus. Hij heeft een demo gegeven over een game die ook te vinden is tussen de tutorials van Unity en van thuis uit kan volgen. De tutorial noemt Adventure Game Tutorial, Deze tutorial was voor mensen die al redelijk bekend zijn met Unity en 3D game design in het algemeen. In deze tutorial werd er gebruik gemaakt van een inventory, hierin kan je objecten bijhouden en deze achteraf gebruiken in het spel, en een simpel contextsysteem om te interageren met andere objecten in de scene.

De bedoeling van de adventure game was dat we met ons personage bepaalde acties uitvoerden zodat we voorbij een bewaker konden geraken. In de scene kon men ook interageren met een aantal objecten om meer informatie hierover te verkrijgen. Om alle acties uit te voeren moest het personage naar verschillende ruimtes lopen. Er waren in totaal twee ruimtes: het atrium waar de bewaker zich bevond en een markt waar we spullen moesten zoeken om ons te vermommen. Als we alle spullen hadden gevonden konden we voorbij de bewaker geraken en was het spel gedaan. Dit is helaas niet volledig gelukt aangezien we maar één sessie hadden en deze tutorial toch wel veel tijd kost om volledig te maken. Hiernaast heb ik zelf nog wel wat geprobeerd tijdens de pauzes en heb ik een aantal personages toegevoegd aan de scene om deze wat levendiger te maken.

Buiten de sessies heb ik me ook nog beziggehouden met een klein project om aan mezelf te tonen wat ik uit de sessies en mijn eigen kennis van Unity heb geleerd. In dit project heb ik een bestuurbaar “first person character” en een omgeving gemaakt bestaande uit bergen en bossen. De bedoeling was om een soort van horror game te ontwikkelen waarbij een grote groep zombies je eigen character achtervolgen. Hier heb ik ook uit mezelf geleerd hoe ik moest werken met belichting, omgevingen en “pathfinding” in een 3D game engine.

Reflectie

Ik heb me ten eerste voor dit seminarie ingeschreven omdat mijn vriendengroep dacht dat dit wel een interessant seminarie kon zijn, maar hiernaast ben ik zelf ook zeer geïnteresseerd in alles dat te maken heeft met game design en het game development proces. Ook had ik al wat ervaring met Unity aangezien ik het thuis al had geprobeerd en vele tutorials heb gevolgd, mijn kennis over Unity en game design lag dus ook wat hoger dan de meesten tijdens deze sessies, waardoor ik sneller klaar was en extra zaken begon toe te voegen aan de projecten zoals de extra personages in het “Adventure game tutorial” project. Hierna heb ik om mezelf te testen dus ook een apart project ontwikkelt.

Tijdens deze sessies is mijn kennis van C# zelf niet zo heel hard verbeterd, de meeste dingen die we moesten doen waren vrij simpel of had ik al eens gedaan thuis. De code die wij geschreven hebben komen ook niet overeen met code die tijdens de lessen Programming Advanced of dergelijke worden gezien. Deze sessies zijn een uitbreiding voor game development zodat je toch de basis meekrijgt en hier in de toekomst sneller complexe delen rond Unity kan aanleren.

De sessies zelf waren wel heel interessant en gaven me meer inzicht in hoe het maken van een game nu precies verloopt. Zelf zou ik wel niet willen verdergaan in het programmeren van games en focus ik me liever op webtechnologieën.

De reden waarom ik deze activiteit heb gekozen is omdat dit me een interessant onderwerp leek, ik speel vaak games en ben altijd al een fan geweest op dat vlak. De sessies zelf zag ik meer als een uitbreiding van mijn kennis op vlak van Unity dan een instap tot game development, aangezien ik er al mee had gewerkt. Ik zou deze sessies zeker aan andere studenten aanraden, zeker als je van plan bent om richting game development te gaan.
